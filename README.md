Trading Algorithm with Hawkes BSI and Multinomial Logistic Regression

This algorithm predicts short-term price moves using Hawkes Buy/Sell Imbalance (BSI) and a multinomial logistic regression model. It loads TAQ trade data, filters and resamples it, then computes BSI to capture short-term order flow pressure. Additional features like short-term returns, volatility, and price deviation from moving averages are calculated and fed into the logistic regression model to predict whether the price will go up, down, or stay flat. Trading signals are generated when predicted probabilities exceed a threshold, opening long or short positions accordingly. Positions are held until the BSI crosses zero, indicating a reversal in order flow. The algorithm can simulate realistic execution with latency, slippage, partial fills, and borrowing costs, recording each trade’s entry, exit, duration, and profit or loss. The algorithm requires TAQ data with columns DATE, TIME_M, PRICE, SIZE, TR_CORR, and TR_SCOND. 

NYSE TAQ data is not included because it is proprietary and licensed by the exchange, so users must provide their own copies or sample datasets for testing.
